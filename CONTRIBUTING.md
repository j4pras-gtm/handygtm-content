# Contributing

Thanks for considering a contribution. This repo holds the editorial content for HandyGTM.

## Before you open a PR

- Read [`EDITORS_GUIDE.md`](EDITORS_GUIDE.md).
- Match the existing tone — direct, operator-first, no hype, no keyword stuffing.
- One topic per file. One file per URL.
- Every playbook links to ≥ 2 related pages.
- Every blog post links to ≥ 1 playbook.

## Frontmatter checklist

- [ ] `slug` matches the filename
- [ ] `title` is ≤ 70 characters
- [ ] `summary` / `excerpt` is ≤ 160 characters
- [ ] `seo.metaTitle` is ≤ 60 characters
- [ ] `seo.metaDescription` is ≤ 155 characters
- [ ] `tools:` slugs all exist in `tools/`

## Review

A maintainer will review within a few days. On merge, the site rebuilds automatically.
