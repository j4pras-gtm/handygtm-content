## Why this playbook

Build a continuously refreshed ICP list using firmographic, technographic, and behavioral signals.

## The system

A repeatable, low-overhead system for the **outbound** motion. Designed for lean teams that want output without the operational drag.

## What you need

- apollo
- clay
- apify

## Steps

1. **Define the goal** — write the outcome in one sentence. If you can't, the playbook isn't ready.
2. **Set up the tools** — connect everything end-to-end. Test before you scale.
3. **Run a small first batch** — measure, adjust, then scale.

## Common failures

- Skipping the goal step and "just running it"
- Over-tooling before the system is proven
- Scaling before the first batch produces clean signal
